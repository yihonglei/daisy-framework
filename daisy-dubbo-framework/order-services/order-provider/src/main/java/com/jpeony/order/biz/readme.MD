> 订单交易这块，我写了一个执行引擎，基于pipeline模式的设计。

# 模块划分
* callback  回调
* context   上下文信息
* convert   上下文转化
* factory   工厂，用来构建pipeline
* handler   具体的业务处理

涉及到复杂逻辑处理的板块，都可以基于这个引擎来进行扩展

包括营销模块、支付模块
